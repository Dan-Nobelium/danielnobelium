Overview

As Software Project Manager for the University of New England (UNE) from March 2024 to June 2024, I developed a comprehensive Excel-based solution to help researchers identify eligible journals under various Read and Publish agreements. This tool consolidates data from multiple sources and provides an easy-to-use interface for researchers to determine which publishing agreements apply to specific journals.

The solution streamlined journal management workload from 40 hours down to just 2, reducing publication research time by 80%. It's now used on the official UNE website, where researchers can download the tool to check journal eligibility for their publishing needs.

Challenge

UNE faced significant challenges in helping researchers navigate publishing options:

  • Data about eligible journals was scattered across multiple sources with no unified system
  • Information from the Council of Australian University Librarians (CAUL), Excellence in Research for Australia (ERA), and Scimago Journal Rankings needed integration
  • Regular updates were required as publisher agreements changed and new journals were added
  • Researchers needed simple access to determine journal eligibility for publishing agreements
  • The solution needed to be maintainable by UNE library staff without specialized technical knowledge
  • Manual data processing was taking staff approximately 40 hours per update cycle

Approach

I implemented a comprehensive solution using PowerQuery and Excel:

  1. Created an automated data processing system that pulls from multiple data sources:
    • CAUL Read & Publish Agreements (.xlsx format)
    • ERA Submission Journal List (.xlsx format)
    • Scimago Journal Rankings (.csv format)
  2. Developed a file monitoring system to automatically detect the latest data files
  3. Implemented robust data cleaning and transformation processes to handle inconsistencies
  4. Created a consolidated database with proper relationships between data sources
  5. Built an automated document generation system that creates a clean, user-friendly Excel file
  6. Designed detailed documentation for UNE staff to maintain the system
  7. Established a one-click solution for processing scientific journal data, eliminating manual data cleaning processes

PowerQuery Data Integration Pipeline


// Main file detection and processing logic for CAUL data
let
    // Gets all files in the CAUL data directory
    Source = Folder.Files(FILENAME & "\data\CAUL"),

    // Filtered out non-relevant files
    #"Filtered rows" = Table.SelectRows(Source, each not Text.StartsWith([Name], "~$") and [Extension] = ".xlsx" and [Attributes]?[Hidden]? <> true),

    // Sort rows by most recently modified
    #"Sorted rows" = Table.Sort(#"Filtered rows", {{"Date modified", Order.Descending}}),

    // Keep the top row
    #"Kept top rows" = Table.FirstN(#"Sorted rows", 1),

    // Opens the workbook and gets the data from the desired table
    #"Open workbook" = Table.SelectRows(Table.RemoveColumns(Table.AddColumn(#"Kept top rows", "Data", each Excel.Workbook(File.Contents([Folder Path] & [Name]))), {"Folder Path", "Attributes", "Date created", "Date modified", "Date accessed", "Extension", "Name", "Content"}){0}[Data], each [Kind] = "Sheet" and [Hidden] <> true){0}[Data],

    // Uses the first row as the table headers
    #"Promoted Headers" = Table.PromoteHeaders(#"Open workbook", [PromoteAllScalars=true]),

    // Remove any rows where the journal name is null
    #"Remove null journal rows" = Table.SelectRows(#"Promoted Headers", each [Journal Name] <> null),

    // Applies the correct data types to each column
    #"Changed Type" = Table.TransformColumnTypes(#"Remove null journal rows",{{"Journal Name", type text}, {"Included in CAUL OA agreement?", type text}, {"Notes", type text}, {"Publisher Name", type text}, {"Last Updated", type date}, {"ISSN", type text}, {"eISSN", type text}, {"Journal Website", type text}, {"Journal Type", type text}, {"Journal Abbreviation", type text}, {"Imprint Name", type text}, {"Included in CAUL discount scheme?", type text}}),

    // Conditionally adds the string "Included in CAUL discount scheme" to the Notes column
    #"Add discount scheme note" = Table.TransformColumns(Table.ReplaceValue(Table.ReplaceValue(#"Changed Type", null, " ", Replacer.ReplaceValue, {"Notes"}), each [Notes], each if [#"Included in CAUL discount scheme?"] = "Yes" then [Notes] & " Included in CAUL discount scheme" else [Notes], Replacer.ReplaceText, {"Notes"}), {{"Notes", Text.Trim, type text}}),

    // Condenses all ISSN for a single journal into one row
    #"Unique rows by ISSN" = Table.ExpandListColumn(Table.TransformColumns(Table.TransformColumns(Table.CombineColumns(#"Add discount scheme note", {"ISSN", "eISSN"}, Combiner.CombineTextByDelimiter(" ", QuoteStyle.None), "ISSN"), {{"ISSN", Text.Trim, type text}}), {{"ISSN", Splitter.SplitTextByDelimiter(" ", QuoteStyle.Csv), let itemType = (type nullable text) meta [Serialized.Text = true] in type {itemType}}}), "ISSN"),

    // Reorders the columns
    #"Reordered Columns" = Table.ReorderColumns(#"Unique rows by ISSN",{"Journal Name", "ISSN", "Included in CAUL OA agreement?", "Notes", "Publisher Name", "Last Updated", "Journal Website", "Journal Type", "Journal Abbreviation", "Imprint Name", "Included in CAUL discount scheme?"}),

    // Remove unrequired columns
    #"Removed Columns" = Table.RemoveColumns(#"Reordered Columns",{"Last Updated", "Journal Website", "Journal Abbreviation", "Imprint Name", "Included in CAUL discount scheme?"}),

    // Rename columns to match original tool
    #"Renamed Columns" = Table.RenameColumns(#"Removed Columns",{{"Included in CAUL OA agreement?", "Included in R&P agreement"}}),

    // Replace null values with empty strings
    #"Replaced Value" = Table.ReplaceValue(#"Renamed Columns",null,"",Replacer.ReplaceValue,{"Journal Type"})
in
    #"Replaced Value"

Final Data Consolidation


// Final Data Integration - Joining CAUL, ERA, and SCIMAGOJR data
let
    // First join (left join ERA to CAUL data)
    Source = Table.NestedJoin(CAUL, {"ISSN"}, ERA, {"ISSN"}, "ERA", JoinKind.LeftOuter),

    // Second join (left join SCIMAGOJR to CAUL data)
    #"Merged Queries" = Table.NestedJoin(Source, {"ISSN"}, SCIMAGOJR, {"Issn"}, "SCIMAGOJR", JoinKind.LeftOuter),

    // expand the merged data
    #"Expanded ERA" = Table.ExpandTableColumn(#"Merged Queries", "ERA", {"FoR Code/Name"}, {"FoR Code/Name"}),
    #"Expanded SCIMAGOJR" = Table.ExpandTableColumn(#"Expanded ERA", "SCIMAGOJR", {"SJR", "SJR Best Quartile", "H index", "Categories", "Areas"}, {"SJR", "SJR Best Quartile", "H index", "Categories", "Areas"}),

    // Sort the journals by lexicographical order
    #"Sorted Rows" = Table.Sort(#"Expanded SCIMAGOJR",{{"Journal Name", Order.Ascending}}),

    // Groups the data by the journal name
    #"Grouped Rows" = Table.Group(#"Sorted Rows", {"Journal Name"}, {{"ISSN", each Text.Combine([ISSN], "; "), type nullable text}, {"Included in R&P agreement", each List.Max([#"Included in R&P agreement"]), type nullable text}, {"Notes", each List.Max([Notes]), type nullable text}, {"Publisher Name", each List.Max([Publisher Name]), type nullable text}, {"Journal Type", each List.Max([Journal Type]), type nullable text}, {"FoR Code/Name", each List.Max([#"FoR Code/Name"]), type nullable text}, {"SJR Best Quartile", each List.Max([SJR Best Quartile]), type nullable text}, {"SJR", each List.Max([SJR]), type nullable number}, {"H index", each List.Max([H index]), type nullable number}, {"SJR Categories", each List.Max([Categories]), type nullable text}, {"SJR Areas", each List.Max([Areas]), type nullable text}})

Automated Export Generation


' VBA code to generate the final document
Sub GenerateDocument()
'
' Generates a copy of the refreshed data to the current directory
'
    Utils.RefreshIndicator ("B2")
    Utils.RefreshConnections
    Utils.RefreshIndicator ("B2")
    ThisWorkbook.Save ' Save the current file before creating a new one
    Utils.DeleteQueries
    Utils.DeleteSheet ("Control Panel")
    Utils.SaveCopy
End Sub

Sub SaveCopy()
'
' Saves the workbook in the current directory with a new name
'
    Application.DisplayAlerts = False ' disable alerts to prevent the user needing to click okay to saving a non-macro file
    
    fname = Format(Date, "yyyy-mm-dd") & " Read and Publish Journal Tool - UNE.xlsx" ' A string containing the desired filename with optimal date format for sorting
    ActiveWorkbook.SaveAs ThisWorkbook.Path & "\" & fname, xlWorkbookDefault
    
    Application.DisplayAlerts = True
End Sub

Demo

The Read and Publish Journal Tool helps UNE researchers easily determine if a journal is covered by an existing agreement:

System Workflow

  1. Users download the tool from the UNE website
  2. They open the Excel file and check the "Read me tips" tab first
  3. They use the search or filter functionality to find specific journals
  4. The tool displays eligibility status for various publishing agreements
  5. Researchers can make informed decisions about where to publish

Maintenance Workflow

  1. UNE staff place updated data files in the appropriate folders
  2. They open the Database tool and click "Generate New Document"
  3. The system automatically processes all data and creates a new Excel file
  4. The new file is uploaded to the UNE website for researchers to access
  5. The entire process takes about 2 hours, compared to the previous 40-hour manual workflow

Outcomes

Key Achievements

  • Successfully implemented on the official UNE website for campus-wide use
  • Consolidated over 30,000 journal entries from multiple publishers
  • Reduced data update process from days to minutes
  • Streamlined journal management workload from 40 hours down to just 2 hours
  • Reduced publication research time by 80% for researchers
  • Eliminated manual cross-referencing errors
  • Created a maintainable solution with comprehensive documentation
  • Provided researchers with clear guidance on eligible publishing options

Value Delivered

This tool has transformed how UNE researchers approach journal selection and publishing decisions. It has helped scholars maximize institutional resources by identifying opportunities to publish in open access journals at no additional cost through the university's existing agreements.

The automation has freed up significant staff time that was previously spent manually updating and cross-referencing journal lists, allowing library staff to focus on providing direct support to researchers. The one-click solution for processing scientific journal data eliminated the need for manual handling and formatting of data being imported from scientific journal aggregation services.
